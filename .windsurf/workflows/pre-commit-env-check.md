# 提交代码前的环境配置检查

此工作流用于在提交代码前检查环境配置文件，确保各环境配置独立且正确。

## 检查目标

确保以下文件中只包含对应环境的配置：
- `application-dev.yml` - 只包含dev相关配置
- `application-qa.yml` - 只包含qa相关配置  
- `application-prod.yml` - 只包含prod相关配置

## 执行方式

直接在对话中请求检查：
```
请检查环境配置文件中是否有其他环境的配置混入
```

## 检查内容

### 1. 环境关键词检查
检查配置值中是否包含其他环境的关键词：
- **dev关键词**: dev, development, local
- **qa关键词**: qa, test, staging
- **prod关键词**: prod, production, live

### 2. 常见错误模式
- prod文件中包含dev/qa的数据库URL
- dev文件中包含prod的邮件配置
- qa文件中包含local的文件路径
- 任何环境文件中包含其他环境的标识符

## 使用方法

1. **提交前检查**: 在提交代码前，请求Cascade检查环境配置
2. **代码审查**: 在代码审查过程中，可以要求检查环境配置
3. **定期检查**: 定期执行检查确保配置文件质量

## 示例检查请求

```
请检查以下环境配置文件是否有配置错误：
- application-dev.yml
- application-qa.yml  
- application-prod.yml

重点检查：
1. prod环境是否混入了dev/qa配置
2. 各环境配置值是否包含其他环境的关键词
3. 邮件组配置是否正确对应环境
```

## 注意事项

- 此检查仅通过文本分析进行，不执行代码
- 检查结果基于关键词匹配和模式识别
- 如有疑问，请人工复核具体的配置项
