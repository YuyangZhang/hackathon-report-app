# Global Language Rules

## Language Usage Policy

### Code Language
- All code, variable names, function names, class names, file names, and technical identifiers must be in English
- Source code files, configuration files, documentation files should use English for technical content
- Windsurf-generated rules, workflows, and automation scripts must use English for technical content and structure

## Implementation Guidelines

1. **Communication**: Always respond to users in Chinese
2. **Code Generation**: Generate code with English names and syntax
3. **File Creation**: Use English for file names and technical content
4. **Comments**: Add English comments to explain code logic
5. **Error Handling**: Provide English error messages
6. **Constants**: Do not modify existing constant strings defined in code

## Purpose

This policy ensures clear communication with Chinese-speaking users while maintaining international coding standards and best practices for code readability and collaboration.
