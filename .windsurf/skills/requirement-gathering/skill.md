# Requirement Gathering Skill

This skill helps you collect and capture requirements from files stored in the jira folder.

## Input
- **file name**: Name of the file in the jira folder to analyze

## Process

1. **Read the file content** from the jira folder with file name is **file name**
2. **Collect requirements** by analyzing the content for:
   - Functional requirements
   - Non-functional requirements  
   - User stories
   - Acceptance criteria
   - Business requirements
   - Technical requirements
   - Constraints and assumptions

## Output
- Structured collection of all identified requirements
- Categorized by requirement type
- Ready for requirement clarification phase
- put the output in file folder jira/**file name**/requirements.json

## Usage

Use this skill when you need to:
- Extract requirements from JIRA tickets or documentation
- Begin the requirement gathering process
- Structure unstructured requirement information

The skill will help you transform raw JIRA content into organized, actionable requirements that can be further clarified and refined.
